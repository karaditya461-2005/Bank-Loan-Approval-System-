1. PROJECT OVERVIEW & OBJECTIVES

Problem Statement: Banks receive thousands of loan applications requiring manual review, leading to slow processing, inconsistent decisions, and high operational costs. This project develops an automated ML system to predict loan approval with high accuracy.

Solution: A supervised machine learning classification model that analyzes 11 customer features to predict loan approval status (Approved/Rejected) in real-time, achieving 83% accuracy with Random Forest algorithm.

Business Impact: Processes applications in <1 second (vs 2-5 days manual review), reduces cost from $50 to $0.10 per application, maintains consistency, and scales to unlimited throughput.

7. WHY F1-SCORE IS THE OPTIMAL METRIC

F1-Score = 2 √ó (Precision √ó Recall) / (Precision + Recall)

Rationale for Loan Approval:

  • Balanced Evaluation: Considers both false positives (approving bad loans = financial loss) and false negatives (rejecting good customers = lost revenue)
  • Class Imbalance Handling: Dataset has 69% approvals vs 31% rejections. Accuracy alone would be misleading.
  • Business Cost Alignment: Both error types are costly but differently. Precision (84%) minimizes risky approvals. Recall (77%) maximizes customer capture.
  • Industry Standard: F1-Score is widely used in financial risk modeling for balanced decision-making.
Alternative Scenarios: Use Precision if bad loans are catastrophic (subprime crisis scenario). Use Recall if competition is fierce (must approve all good customers). F1-Score balances both for typical banking operations.
